Getting started: where it runs, working folder, what's included

Where does this run?

On your computer — Linux, macOS, or WSL on Windows — in a bash terminal. You clone the GitHub repo, install a conda environment (or put the tools on PATH), then run shell scripts locally. This web page is only the guide; it does not execute anything.

Do I need a working folder?

Yes. Clone the repo anywhere — that directory is your project root. You run commands from specific subfolders; paths in the settings below are paths on your machine (relative to where you run the script, or absolute). The pipeline writes outputs to OUT_DIR, which it creates if missing — you choose that folder. 

$HOME/example_workflow/           ← git clone (project root)
├── pipeline/                     ← scripts (stage 00–07, run_pipeline.sh)
├── test_case/                    ← demo: bash run_demo.sh from here
├── panels/                       ← example gene panel (shipped)
└── test_case/results/            ← demo OUT_DIR (created on first run)

/data/genomics/                   ← recommended for real WGS (outside repo)
├── refs/hg38.fa                  ← REF_FASTA — large, download once
├── clinvar/clinvar_grch38.vcf.gz ← CLINVAR_VCF — download once
└── fastq/sample_R1.fastq.gz      ← your sequencer output

Tip: keep multi-gigabyte reference files outside the git repo. Point REF_FASTA and CLINVAR_VCF at them with absolute paths, or export those variables before running (see stage 00).

Start here: demo first, real data second

  1. Clone and install tools (once)
    This installs bwa, samtools, bcftools, and fastp — the tools the demo actually runs. It does not install reference FASTAs, ClinVar, or snpEff. GATK4 and snpEff are optional (secondary caller and gene annotation). Do not verify with bare bwa / samtools — those commands only print usage help. Use pipeline/verify_tools.sh.
  2. Run the built-in demo (no extra downloads)
    Run from $HOME/example_workflow (the clone from step 1). Always use bash run_demo.sh. Outputs land in test_case/results/. If run_demo.sh prints an error, fix that first — do not run check_demo.py until the demo finishes.
  3. Move to real data (when ready) — download reference + ClinVar (see inputs guide), put your FASTQs somewhere, write a tab-separated samples.tsv (no header: sample_id, case|control, R1, R2). From the repo root: bash pipeline/run_pipeline.sh samples.tsv. To re-run the shipped demo cohort instead: bash pipeline/run_pipeline.sh test_case/samples.tsv.

Three kinds of settings — file path vs tool name vs your choice

Knowing snpEff as a tool is not the same as knowing what to type in the pipeline config. Most confusion is mixing up files you point at with names that tell a tool which built-in resource to use.

Kind What you type Who owns the data? Examples
File path Path to a file on disk You download or create the file; the pipeline reads it by path REF_FASTA, CLINVAR_VCF, FASTQ paths, PANEL_BED, MANIFEST_TSV
Tool lookup name A short name, not a path The tool finds data in its own install location after you run its download command SNPEFF_DB → after optional conda install -c bioconda snpeff: snpEff databases then snpEff download <name>. BCFTOOLS_PLOIDY → bcftools call --ploidy ?.
Setting you choose Folder, number, or label No download — you pick a writable directory or display name OUT_DIR, TMP_DIR, THREADS, PANEL_NAME, sample_id

Input reference: each variable in detail

Quick lookup table. Per-stage fields below repeat the same explanations. All reference variables must use the same genome build (GRCh37, GRCh38, T2T, etc.).

Variable Kind Default / demo Where and how to get it (real data)
REF_FASTA File path test_case/refs/panel_region.fa (tiny slice, in repo) Download whole-genome FASTA for your build: UCSC hg38, GATK bundle, or T2T-CHM13. Run stage 01 once to build indexes (bwa index, etc.).
CLINVAR_VCF File path test_case/refs/clinvar_panel_subset.vcf.gz (1 variant, in repo) wget from NCBI — vcf_GRCh38/clinvar.vcf.gz or vcf_GRCh37/. Must match REF_FASTA build. Stage 01 runs tabix if present.
SNPEFF_DB Tool lookup name GRCh38.mane.1.2.ensembl — not installed unless you run download Optional. First: conda install -c bioconda snpeff. Then snpEff databases | grep -i GRCh38 and snpEff download <name>. Type that name here. Demo works without snpEff (ClinVar-only annotation).
BCFTOOLS_PLOIDY Tool lookup name GRCh38 (built into bcftools) No download. List presets with bcftools call --ploidy ?. Match your reference: GRCh38 for hg38, GRCh37 for hg19.

Roadmap

  • nf-core/sarek — alternative sequencing engine (Nextflow); clinical stages 05–07 shared
  • Array / PLINK ingestion path (documented, not built)
  • Pluggable callers (DeepVariant) in bash pipeline
  • Linear reference only (no DRAGEN pangenome graphs) — any build via REF_FASTA (GRCh37, GRCh38, T2T, …); demo defaults to GRCh38
  • Not validated for clinical diagnostics
